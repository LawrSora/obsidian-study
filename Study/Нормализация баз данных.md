    <div class = "boring-div" ><b>Цель нормализации</b>:<br>
	   - Устранение избыточности за счёт декомпозиции отношений.<br>
	   - В каждом отношении должны храниться только первичные факты (не выводимые из других фактов). </div>
	<div class = "boring-div"><b>Нормальные формы</b>:<br>
	   - 1 НФ (Первая нормальная форма)<br>
	   - 2 НФ (Вторая нормальная форма)<br>
	   - 3 НФ (Третья нормальная форма)<br>
	   - НФ Бойса-Кодда (усиленная 3 НФ)<br>
	   - 4 НФ, 5 НФ, 6 НФ, Доменно-ключевая НФ</div>

### 1 НФ (Первая нормальная форма)
<div class = "boring-div">
<b>Определение:</b><br>
Переменная отношения находится в 1НФ, если в любом допустимом значении отношения каждый его кортеж содержит только одно значение для каждого из атрибутов.</div>
<div class = "green-div">Проще:<br>
- В каждой ячейке таблицы должно быть только одно значение.<br>
- Не должно быть повторяющихся строк.
</div>

### 2 НФ (Вторая нормальная форма)
<div class="boring-div">
<b>Определение:</b><br>
Переменная отношения находится во 2НФ, если она находится в 1НФ, и каждый неключевой атрибут неприводимо (функционально полно) зависит от её потенциального ключа.
</div>

<div class="green-div">
<b>Проще:</b><br>
- Таблица должна быть в 1НФ.<br>
- Должен быть первичный ключ.<br>
- Все атрибуты должны зависеть от <b>всего</b> первичного ключа, а не от его части.
</div>

### 3 НФ (Третья нормальная форма)
<div class = "boring-div">
<b>Определение:</b><br>
Переменная отношения находится в 3НФ, если она находится во 2НФ, и отсутствуют транзитивные функциональные зависимости неключевых атрибутов от ключевых.
</div>

<div class = "green-div">Проще:<br>
- Убедитесь, что таблица во 2НФ<br>
- Удалите атрибуты, которые зависят от других неключевых атрибутов (а не от первичного ключа)<br>
- «Каждый факт должен говорить только о ключе, всей правде о ключе и ничего кроме ключа»
</div>

### НФ Бойса-Кодда (BCNF)
<div class = "boring-div">
<b>Определение:</b>
Отношение находится в BCNF если для каждой нетривиальной функциональной зависимости X → Y, X является суперключом.</div>

<div class = "green-div">Проще:<br>
- Усиленная версия 3НФ<br>
- Все зависимости атрибутов от ключа должны быть прямыми<br>
- Не допускается зависимость части ключа от неключевых атрибутов
</div>

---

Пример нарушения BCNF:
<div class = "boring-div">
<table border="1">
  <tr><th>Преподаватель</th><th>Предмет</th><th>Аудитория</th></tr>
  <tr><td>Иванов</td><td>Математика</td><td>301</td></tr>
  <tr><td>Петров</td><td>Физика</td><td>215</td></tr>
  <tr><td>Иванов</td><td>Физика</td><td>215</td></tr>
</table>
<p>Зависимость: Аудитория → Предмет (одна аудитория закреплена за предметом)</p>
</div>

---

Исправленная версия:
<div style="display: grid;grid-template-columns: 1fr 1fr;gap: 15px;">
  <div class = "boring-div">
    <b>Преподаватели-Аудитории</b><br>
    <table border="1">
      <tr><th>Преподаватель</th><th>Аудитория</th></tr>
      <tr><td>Иванов</td><td>301</td></tr>
      <tr><td>Петров</td><td>215</td></tr>
    </table>
  </div>
  
  <div class = "boring-div">
    <b>Аудитории-Предметы</b><br>
    <table border="1">
      <tr><th>Аудитория</th><th>Предмет</th></tr>
      <tr><td>301</td><td>Математика</td></tr>
      <tr><td>215</td><td>Физика</td></tr>
    </table>
  </div>
</div>

### 4 НФ (Четвертая нормальная форма)
<div class = "boring-div">
<b>Определение:</b>
Отношение находится в 4НФ если в нём отсутствуют нетривиальные многозначные зависимости, не следующие из потенциальных ключей.
</div>

<div class = "green-div">Проще:<br>
- Таблица не должна содержать независимые группы многозначных зависимостей<br>
- Если значение A определяет множество значений B и C независимо друг от друга - требуется декомпозиция
</div>

---

Пример нарушения 4НФ:
<div class = "boring-div">
<table border="1">
  <tr><th>Студент</th><th>Курс</th><th>Хобби</th></tr>
  <tr><td>Иван</td><td>Математика</td><td>Шахматы</td></tr>
  <tr><td>Иван</td><td>Физика</td><td>Футбол</td></tr>
  <tr><td>Мария</td><td>Химия</td><td>Рисование</td></tr>
</table>
<p>Курсы и хобби студента независимы друг от друга</p>
</div>

### 5 НФ (Пятая нормальная форма)
<div class = "boring-div">
<b>Определение:</b>
Отношение находится в 5НФ если каждая нетривиальная зависимость соединения определяется потенциальными ключами отношения.
</div>

<div class = "green-div">Проще:<br>
- Таблицу нельзя декомпозировать без потери информации<br>
- Все сложные связи должны быть представлены через естественное соединение
</div>

---

Пример нарушения 5НФ:
<div class = "boring-div">
<table border="1">
  <tr><th>Поставщик</th><th>Деталь</th><th>Проект</th></tr>
  <tr><td>A</td><td>Болт</td><td>X</td></tr>
  <tr><td>A</td><td>Гайка</td><td>Y</td></tr>
  <tr><td>B</td><td>Винт</td><td>X</td></tr>
</table>
<p>Связь между тремя сущностями требует тройного соединения</p>
</div>

---

Декомпозиция на 5НФ:

### Доменно-ключевая НФ (DKNF)
<div class = "boring-div">
<b>Определение:</b>
Отношение находится в DKNF если каждое ограничение на отношение является логическим следствием ограничений доменов и ключевых ограничений.
</div>

<div class = "green-div">Проще:<br>
- Идеальная нормальная форма (теоретическая)<br>
- Все бизнес-правила реализованы через ключи и домены<br>
- Практически не достижима в реальных проектах
</div>

---

<div class = "green-div">
<b>Итоговые рекомендации:</b><br>
4. Для большинства систем достаточно 3НФ или BCNF<br>
5. Высшие нормальные формы требуются для специальных случаев<br>
6. Денормализация допустима для оптимизации производительности<br>
7. Всегда учитывайте компромисс между нормализацией и практической эффективностью
</div>
