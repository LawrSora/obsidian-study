# №1
### **Типы данных в PHP**

**В PHP есть 12 типов данных**: 

1.    **bool** (логический тип);

Переменные логического типа могут принимать два значения: `true` и `false` или иначе говоря `истина` и `ложь`. Чаще всего логические значения используются в условных конструкциях:

```php
$foo = true;
$boo = false;
```

2.     int (целые числа); 

Представляет целое число со знаком.

```php
$num = -100;
echo $num;
```

Кроме десятичных целых чисел PHP обладает возможностью использовать также двоичные, восьмеричные и шестнадцатеричные числа. Шаблоны чисел для других систем:

·        шестнадцатеричные : 0 \[xX]\[0-9a-fA-F]

·        восьмеричные : 0\[0-7]

·        двоичные : 0b\[01]

Например:

```php
$num_10 = 28;
$num_2 =  0b11100;// двоичное число
$num_8 = 034;// восьмеричное число
$nim_16 = 0x1C;// шестнадцатеричное число
```

Переменная типа `int` занимает в памяти 32 бита, то есть может принимать значения от -2 147 483 648 до 2 147 483 647. Если переменная получает числовое значение вне этого диапазона, то она трактуется как переменная типа `float`

3.    **float** (дробные числа); 

Размер числа с плавающей точкой зависит от платформы. Максимально возможное значение, как правило, составляет 1.8*10308 с точностью около 14 десятичных цифр. Например:

```php
$a1 = 1.5;
$a2 = 1.3e4;//1.3 * 10^4
$a3 = 6E-8;//0.00000006
```

4.    **string** (строки); 

Для работы с текстом можно применять строки. Строки бывают двух типов: в двойных кавычках и одинарных. От типа кавычек зависит обработка строк интерпретатором. Так, переменные в двойных кавычках заменяются значениями, а переменные в одинарных кавычках остаются неизменными.

```php
$a = 10;
$b = 5;
$result = " $a + $b  <br>";
echo $result;
$result = ' $a + $b ';
echo $result;
```

В этом случае мы получим следующий вывод:

5.    10+5 

6.    $a + $b

Кроме обычных символов, строка может содержать специальные символы, которые могут быть неправильно интерпретированы. Например, нам надо добавить в строку кавычку:

```php
$text = "Модель"Apple II"";
```

Данная запись будет ошибочна. Чтобы исправить ошибку, мы можем сочетать различные типы кавычек ('Модель "Apple II"' или "Модель 'Apple III'") или использовать слеш, чтобы ввести кавычку в строку:

```php
$text = "Модель \"Apple II\"";
```

7.    **array** (массивы); 

Массив (array) создают языковой конструкцией array(). В качестве аргументов она принимает любое количество разделённых запятыми пар _ключ_ `=>` _значение_.

array(_key_  => _value_, key2 => value2, key3 => value3, ...)

Запятая после последнего элемента массива необязательна и её можно опустить. Обычно это делается для однострочных массивов, — лучше предпочесть `array(1, 2)` вместо `array(1, 2, )`. Для многострочных массивов, наоборот, обычно указывают висящую запятую, так как упрощает добавление новых элементов в конец массива.

8.    **object** (объекты)

Новые объекты создают выражением new, которое создаёт в переменной экземпляр класса:

```php
<?php  
class foo  
{  
function do_foo()  
{  
echo "Код foo.";  
}  
}   
$bar = new foo;  
$bar->do_foo();  
?>
```

9. **callable** (функции); 

Тип данных callable в PHP используется для передачи функций в качестве аргументов других функций или их возврата. Такие функции могут быть объявлены анонимно или через имя функции. Пример использования callable в качестве аргумента функции:

```php
function multiplyByTwo($num) {  
    return $num * 2;  
}  
function mathOperation($num, callable $callback) {  
    return $callback($num);  
}  
echo mathOperation(5, 'multiplyByTwo'); // Output: 10  
```

Пример использования callable для возврата функции:

```php
function getMultiplier($num) {  
    return function($x) use ($num) {  
        return $num * $x;  
    };  
}  
$multiplier = getMultiplier(5);  
echo $multiplier(3); // Output: 15  
```
Кроме того, тип callable часто используется внутри PHP для передачи callback-функций встроенных функций, таких как array_map или usort.

10. **mixed** (любой тип); 

Тип mixed принимает любое значение. Он эквивалентен объединению типов`object|resource|array|string|float|int|bool|null`. Тип доступен с PHP 8.0.0.

На языке теории типов, mixed — верхний тип. Остальные типы — его подтипы.

11. **resource** (ресурсы); 

Resource — переменная, которая хранит ссылку на внешний ресурс. Ресурсы создают и передают в отдельный набор функций для работы с ресурсами. Список этих функций и типы ресурсов (resource), которые соответствуют функциям, перечисляет приложение.

12. **null** (отсутствие значения). 

Тип null — единичный PHP-тип с единственным значением: null.

Неопределённые и удалённые языковой конструкцией unset() переменные преобразуются в значение null.

Типу null принадлежит только одно значение — регистронезависимая константа null

```php
$var = NULL;  
```

### **Выражения**

Выражения — самые важные строительные элементы PHP. Почти всё, что разработчик пишет в PHP, — выражение. Самое простое и точное определение выражения — «всё, у чего есть значение».

Главные формы выражений — константы и переменные

В записи $a = 5 значение 5 присвоится переменной $a

После этого присвоения ожидается, что значение переменной $a тоже равно 5, поэтому, если написано $b = $a, то значение переменной $b будет = 5

Немного более сложные примеры выражений — функции

```php
function foo ()
{
    return 5;
}
```

$c = foo() то же самое, что и $c = 5. (Функции — это выражения, значение которых — то, что возвращает функция.) Поскольку функция foo() возвращает 5, значением выражения foo() будет 5.

$a = 5 здесь есть два значения — значение целочисленной константы 5 и значение переменной $a, также принимающей значение 5. Но, хотя это неочевидно, здесь есть ещё одно значение — значение самого присвоения. Само присвоение вычисляется в присвоенное значение, в данном случае — в 5. То есть выражение $a = 5, независимо от того, что оно делает, — это выражение со значением 5. Поэтому запись $b = ($a = 5) равносильна записи $a = 5; $b = 5; Поскольку операции присвоения анализируются справа налево, также разрешено написать $b = $a = 5.

Есть префиксный и постфиксный инкремент и декремент.(++$i ++ и --$i --) ++$i изменяет переменную до того как прочесть её значение. $i++  вначале читает значение переменной и только потом изменяет  её.

```php
$i =10;
echo $i++;// 10
echo $i;//11
```

Распространённые типы выражений — выражения сравнения. Эти выражения вычисляются либо как false либо как true. PHP поддерживает операции сравнения >, >= == (равно), != (не равно), < и <= (меньше либо равно). Он также поддерживает операторы строгого равенства: === (равно и одного типа) и !== (не равно или не одного типа). Обычно этими выражениями пользуются в операторах условного выполнения, например, инструкции if.

```php
If($a>$b){
               Echo “мяу”;
}  
```

Смешанные выражения операции и присвоения

Прибавить 3 к текущему значению переменной $a можно, записав $a += 3. суммированное значение переменной $a плюс 3 Поэтому разрешено использовать любой бинарный оператор, например, $a -= 5 (вычесть 5 из значения переменной $a), $b *= 7 (умножить значение переменной $b на 7) и т. д.

==$first ? $second : $third==

Если значение первого подвыражения — true (ненулевое значение), то выполняется второе подвыражение, которое и будет результатом условного выражения. Или выполняется третье подвыражение и его значение будет результатом.

### **Переменные**

Переменные в PHP обозначаются знаком доллара, за которым идёт имя переменной.

Имя переменой должно начинаться с  буквы в диапазонах A-Z или a-z или символа подчёркивания. Дальше можно сколько угодно букв, цифр подчеркиваний.

Неопределённая переменная по умолчанию содержит значение null.

По умолчанию переменные присваиваются по значению. Поэтому когда переменной присваивают выражение, значение исходного выражения копируется в целевую переменную. Это означает, например, что после присваивания значения одной переменной другой переменной, изменение одной переменной не повлияет на другую.

```php
$a = 5 ;  
$b= $a;  
$b = $b -2  
echo $a //5  
echo $b // 3
```

PHP также предлагает другой способ присваивания значений переменным: присваивание по ссылке. Это означает, что новая переменная просто ссылается на оригинальную переменную. Изменения в новой переменной отражаются на оригинале, и наоборот.

Для присваивания по ссылке к началу имени присваиваемой (исходной) переменной просто добавляют амперсанд &. Например, следующий фрагмент кода дважды выводит «Меня зовут Боб»

```php
$foo = 'Боб';            
$bar = &$foo;             
$bar = "Меня зовут $bar";
echo $bar;
echo $foo; 
```

PHP предлагает набор предопределённых переменных, и набор предопределённых массивов, в которых содержатся переменные веб-сервера

Область видимости переменной — контекст, в котором определили переменную. В PHP предусмотрели область видимости функции и глобальную область видимости. Переменная, которую определили за пределами функции, ограничивается глобальной областью видимости.

Переменная, которую создали внутри именованной или анонимной функции, ограничивается областью видимости тела функции. Стрелочные функции вместо этого связывают переменные родительской области видимости и переменные становятся доступны в теле стрелочной функции. При включении файла внутри функции переменные, которые содержатся во включаемом файле, будут доступны так, как будто переменные определили внутри вызывающей функции.

```php
$a = 1; // Определяем переменную $a в глобальной области видимости
function test()
{
    echo $a; // Переменная $a не определена, поскольку конструкция echo указывает на локальную версию переменной $a
}
```

Ключевое(global) слово создаст локальную переменную, которая ссылается на глобальную переменную с тем же названием.

```php
$a = 1;  
$b = 2;  
function Sum()  
{  
global $a, $b;  
$b = $a + $b;  
}  
Sum();  
echo $b;
```

Другая важная особенность области видимости переменной — статическая переменная. Статическая переменная существует только в локальной области видимости функции, но не теряет своего значения, когда выполнение программы выходит из этой области видимости.

Чтобы написать полезную функцию подсчёта, которая не потеряет текущего значения счётчика, переменную $a объявляют статической:

```php
function test()  
{  
static $a = 0;  
echo $a;  
$a++;  
}
```

### **Константы в PHP. Магические константы.**

**Константа — идентификатор, или имя, для простого значения**. Как подсказывает название, это **значение нельзя изменять** во время работы скрипта. Константы чувствительны к регистру. По принятому соглашению, **имена констант пишутся в верхнем регистре**. Правильное **имя начинается с буквы или символа подчёркивания**, за которым следует любое количество букв, цифр и символов подчёркивания. Константу определяют через ключевое слово **const** или функцией [define()](https://www.php.net/manual/ru/function.define.php). Функция [define()](https://www.php.net/manual/ru/function.define.php) разрешает определить константу для произвольного выражения, тогда как у конструкции **const есть ограничения**(допускаются только скалярные выражения ([bool](https://www.php.net/manual/ru/language.types.boolean.php), [int](https://www.php.net/manual/ru/language.types.integer.php), [float](https://www.php.net/manual/ru/language.types.float.php) и [string](https://www.php.net/manual/ru/language.types.string.php)) и массивы ([array](https://www.php.net/manual/ru/language.types.array.php)), которые содержат только скалярные выражения(то есть можно определить одним числом, без направления)). После определения константу нельзя изменить или сделать неопределённой. **Отличия констант от переменных:**

- У констант **нет приставки в виде знака доллара $;**
- Константы разрешается определять и **получать доступ к ним в любом месте** без учёта области видимости;
- Константы **нельзя переопределять или удалять** после первоначального объявления;
- Константам разрешено содержать **только скалярные значения** или массивы.

```php
const CONSTANT = 'Здравствуй, мир.';
echo CONSTANT;
```

**Предопределенные константы -** Эти константы определяет ядро PHP. Сюда входят PHP, движок Zend и SAPI-модули. Например, PHP_VERSION (string)(Текущая версия PHP в виде строки в формате "major.minor.release\[extra\]".)

**Магические константы** – это константы, **значение** которых **изменяется в зависимости от контекста**. Например, значение константы [**__LINE__**](https://www.php.net/manual/ru/language.constants.magic.php#constant.line) зависит от строки скрипта, на которой указали константу(Текущий номер строки файла). На данный момент их 10(__LINE__, __FILE__(Полный путь и имя текущего файла), __DIR__(Каталог файла), __FUNCTION__(Название функции), __CLASS__(Название класса.), __TRAIT__(Имя трейта), __METHOD__(Имя метода класса), __PROPERTY__(Название свойства), __NAMESPACE__(Имя текущего пространства имён), **_ClassName_****::***class**(Абсолютное имя класса)**)**

### Операторы в PHP. Приоритет операторов. Операторы: арифметические, присваивания, сравнения.

Оператором называется нечто, что принимает одно или более значений, и вычисляет новое значение.
Операторы можно сгруппировать по количеству принимаемых ими значений. Унарные операторы принимают только одно значение, например, ! (оператор логического отрицания) или ++ (инкремент). Бинарные операторы принимают два значения; это, например, арифметические операторы + (плюс) и - (минус). Cуществует всего один тернарный оператор, ? :, принимающий три значения.
Если операторы имеют равный приоритет, то будут ли они выполняться справа налево или слева направо — зависит от их ассоциативности. Например, «-» — левоассоциативный оператор. Поэтому, выражение 1 - 2 - 3 сгруппируется как (1 - 2) - 3 и пересчитается в -4. При этом оператор «=» — правоассоциативный, так что выражение $a = $b = $c сгруппируется как $a = ($b = $c).

### Управляющие конструкции в PHP: if, else, elseif/else if

Конструкция `elseif`, как ясно из названия, объединяет конструкции `if` и `else`. Как и конструкция `else`, она расширяет инструкцию `if`, чтобы выполнять другую инструкцию тогда, когда PHP вычисляет выражение начальной конструкции `if` как `false`. Отличие от конструкции `else` состоит в том, что конструкция `elseif` выполнит альтернативное выражение только тогда, когда PHP вычислит условное выражение `elseif` как `true`. Следующий код, в зависимости от значений переменных, будет выводить «a больше b», «a равно b» или «a меньше b»:

```php
<?php
if ($a > $b) {echo "a больше b";
} elseif ($a == $b) {echo "a равно b";} else {
echo "a меньше b";
}
?>
```

Допустимо указывать больше одной конструкции `elseif` в одной инструкции `if`. PHP выполняет первое выражение `elseif`, которое вычисляет как `true`. В PHP допустимо писать `else if` в два слова, тогда поведение будет идентичным конструкции `elseif` в одно слово. Синтаксическое отличие есть (в языке C пишут в два слова), но в итоге оба выражения дадут один и тот же результат.

PHP выполнит инструкцию `elseif` только тогда, когда вычислит предшествующее выражение `if` и предшествующие выражения `elseif` как `false`, а текущее выражение `elseif` — как `true`.

**Замечание**: Заметьте, что записи `elseif` и `else if` будут равнозначны только в синтаксисе с фигурными скобками, как в приведённом примере. В синтаксисе с двоеточием для определения условий `if` и `elseif`, конструкция `elseif` пишется в одно слово. PHP выдаст ошибку синтаксического анализа и завершит работу, если запись конструкции `else if` разделена на два слова.

```php
<?php
/* Некорректный способ: */
if ($a > $b):echo $a . " больше " . $b; 
else if ($a == $b): // Не скомпилируется.  
echo "Строка выше вызывает фатальную ошибку."; 
endif; 
  
/* Корректный способ: */ 
if ($a > $b):echo $a . " больше " . $b; 
elseif ($a == $b): // Заметьте, тут одно слово.  
echo $a . " равно " . $b;
else:  
echo $a . " не больше и не равно " . $b;  
endif;  
?>
```

Инструкция switch похожа на серию инструкций IF, которые проверяют одно и то же выражение. Одну и ту же переменную или выражение часто сравнивают с разными значениями, чтобы выполнить ту часть кода, условию выполнения которой соответствует значение переменной или выражения. Инструкция switch выполняет такие проверки.

```php
switch ($i) {  
case 0:  
echo "Значение переменной \$i равно 0";  
break;  
case 1:  
echo "Значение переменной \$i равно 1";  
break;  
case 2:  
echo "Значение переменной \$i равно 2";  
break;
```

Список инструкций в секциях case разрешается оставлять пустым, тогда управление передаётся списку инструкций следующей секции case.

```php
switch ($i) {  
case 0:  
case 1:  
case 2:  
echo "Значение переменной \$i меньше 3, но не отрицательно";  
break;  
case 3:  
echo "Значение переменной \$i равно 3";  
}
```

Инструкция default управляет случаями по умолчанию. Управление переходит к случаю по умолчанию при несовпадении значения выражения switch со значениями выражений других случаев. Приведём пример:

```php
switch ($i) {  
case 0:  
echo "Значение переменной \$i равно 0";  
break;  
case 1:  
echo "Значение переменной \$i равно 1";  
break;  
case 2:  
echo "Значение переменной \$i равно 2";  
break;  
default:  
echo "Значение переменной \$i не равно 0, 1 или 2";  
}
```

Базовую форму альтернативного синтаксиса получают путём изменения открывающей фигурной скобки на двоеточие :, а закрывающей — на инструкцию  endswitch

```php
switch ($i) :  
case 0:  
case 1:  
case 2:  
echo "Значение переменной \$i меньше 3, но не отрицательно";  
break;  
case 3:  
echo "Значение переменной \$i равно 3";  
endswitch
```

FOR
Цикл for самый сложный цикл в PHP.
Синтаксис цикла for:
```php
for (expr1; expr2; expr3)
    statement
```

 Первое выражение expr1 вычисляется или выполняется только один раз безусловно в начале цикла.
 В начале каждой итерации оценивается выражение expr2. Цикл продолжается, а вложенные инструкции выполняются, если выражение принимает значение true. Выполнение цикла заканчивается, если выражение принимает значение false.
 В конце каждой итерации вычисляется или выполняется выражение expr3.

Каждое из выражений разрешается оставлять пустым или указывать в каждом выражении набор выражений через запятую. В части expr2 каждое выражение, которое отделили запятой, вычисляется, но результат берётся из последнего. Цикл выполняется бесконечно, если выражение expr2 оставили пустым; PHP неявно рассматривает пустое выражение как значение true.

Пример 1

```php
for ($i = 1; $i <= 10; $i++) {
    echo $i;
}
```

Пример 2

```php
for ($i = 1; ; $i++) {
    if ($i > 10) {
        break;
    }

    echo $i;
}

```

#### Альтернативный синтаксис управляющих конструкций

PHP предлагает альтернативный синтаксис для отдельных управляющих структур языка, а точнее: if, while, for, foreach и switch. Для каждой управляющей структуры базовую форму альтернативного синтаксиса получают путём изменения открывающей фигурной скобки на двоеточие :, а закрывающей — на инструкцию, которая соответствует названию структуры: endif;, endwhile;, endfor;, endforeach; или endswitch;.
Пример:
```php
<?php if ($a == 5): ?>
```
A равно 5
```php
<?php endif; ?>
```
В приведённом примере, HTML-блок "A равно 5" вложили в конструкции if, которую записали альтернативным синтаксисом. Блок HTML-разметки отобразится, только если переменная $a равна 5.
Пример 2:
```php
<?php
if ($a == 5):
    echo "Значение переменной $a равно 5";
    echo "...";
elseif ($a == 6):
    echo "Значение переменной $a равно 6";
    echo "!!!";
else:
    echo "Значение переменной $a не равно ни 5, ни 6";
endif;
?>
```
Смешивание синтаксиса в одном и том же блоке управления не поддерживается.
Любой вывод, включая пробельные символы, между инструкцией switch и первой инструкцией case вызовет синтаксическую ошибку. Например, следующий синтаксис недопустим:
```php
<?php switch ($foo): ?>
    <?php case 1: ?>
    ...
<?php endswitch; ?>
```
При этом следующий пример содержит допустимый код, поскольку завершающий символ новой строки после инструкции switch PHP рассматривает как часть закрывающего тега ?> и поэтому между инструкциями switch и case ничего не выводится:
```php
<?php switch ($foo): ?>
<?php case 1: ?>
    ...
<?php endswitch; ?>
```

### Массивы. Виды массивов.

*Массив* в PHP — упорядоченная структура данных, которая связывает _значения_ и _ключи_.

*Массив*  создают языковой конструкцией array(). В качестве аргументов она принимает любое количество разделённых запятыми пар `ключ => значение`.

```php
<php?
$array = array(
"foo" => "bar",
"bar" => "foo",
);
// Работа с коротким синтаксисом массива
$array = [  
"foo" => "bar", 
"bar" => "foo",
];
?>
```

Ключ массива (key) разрешено указывать либо как целочисленное значение (int), либо как строку (string). Значение массива (value) может принадлежать любому типу данных.

В PHP существует три основных типа массивов:

*Индексированные* или *числовые* массивы: массив с числовым ключом(индексом), в котором значения хранятся линейно.

*Ассоциативные* массивы: массив со строковым ключом, в котором вместо линейного хранилища каждому значению может быть назначен определенный ключ.

*Многомерные* массивы: массив, содержащий один или несколько массивов, доступ к которому можно получить через несколько ключей.

Дополнительно произойдут следующие преобразования ключа key:

- Строки (string), содержащие целое число (int) (исключая случаи, когда перед числом указывают знак `+`), будут преобразованы в целое число (int). Например, ключ со значением `«8»` сохранится со значением `8`. При этом, значение `«08»` не преобразуется, так как оно — не корректное десятичное целое.
- Числа с плавающей точкой (float) также преобразуются в целочисленные значения (int) — дробная часть будет отброшена. Например, ключ со значением `8.7` будет сохранится со значением `8`.
- Логический тип (bool) также преобразовывается в целочисленный тип (int). Например, ключ со значением **true** сохранится со значением `1`, а ключ со значением **false** сохранится со значением `0`.
- Тип null преобразуется в пустую строку. Например, ключ со значением `null` сохранится со значением `""`.
- Массивы (array) и объекты (object) _нельзя_ указывать как ключи. Это сгенерирует предупреждение: `Недопустимый тип смещения (Illegal offset type)`.

### Управляющая конструкция foreach. Альтернативный синтаксис foreach.

Управляющая конструкция foreach – это простой способ перебора массивов, она работает только с массивами и объектами (в иных случаях будет выводить ошибку).  
Нам доступно два вида синтаксиса:

1.   foreach (iterable_expression as $value)
statement

2.   foreach (iterable_expression as $key => $value)
statement

Обе конструкции вытаскивают элемент из массива и кидают его в $value, но во втором случаем она еще и показывает ключ этого элемента в массиве с помощью $key

```php
<?php foreach([1, 2, 3, 4] as $value):?>
	<?=$value."<br>"?>
<?php endforeach; ?>>
```

Альтернативный синтаксис используется, потому что код выглядит аккуратным и прекрасно работает автоматическое форматирование. endif, endfor, endforeach прекрасно указывают на то, какой блок был закрыт, а просто скобка порой неясна и надо листать код и смотреть.

 Функции, определяемые пользователем. Анонимные функции.

Анонимные функции в PHP, также известные как замыкания (closures), были введены в PHP 5.3 и позволяют создавать функции без имени. Они могут быть полезны в ситуациях, когда необходимо передать некоторый функционал в качестве аргумента или использовать функцию только в рамках определенного контекста.

### Основные моменты:

1. *Синтаксис*:
   
```php
$func = function($arg1, $arg2) {
	return $arg1 + $arg2;
   };
   echo $func(2, 3); // Вывод: 5
```
   

2. *Использование в качестве аргументов*:
   Анонимные функции часто используются в функциях, которые принимают другие функции в качестве параметров, например, в array_map, array_filter и т.д.
   
```php
$numbers = [1, 2, 3, 4];
   $squared = array_map(function($n) { return $n ** 2; }, $numbers);
```
   

3. *Замыкания*:
   Анонимные функции могут захватывать переменные из внешней области видимости с помощью ключевого слова use. Это позволяет использовать переменные, которые находятся вне функции.
   
```php
$factor = 2;
   $multiply = function($n) use ($factor) {
       return $n * $factor;
   };
   echo $multiply(4); // Вывод: 8
```
   

4. *Область видимости*:
   Анонимные функции имеют свою собственную область видимости, поэтому они не могут обращаться к переменным вне их области видимости, если они не используют use.

5. *Применение*:
   - Создание временных функций для обработки данных
   - Реализация логики обратного вызова (callback)
   - Использование в функциональном программировании

### Пример:
```php
$data = [1, 2, 3, 4, 5];
$filterdData = array_filter($data, function($n) {
    return $n > 2;
}); // $filterdData будет содержать [3, 4, 5]
```

#### Стрелочные функции

Стрелочные функции появились в PHP 7.4 как лаконичный синтаксис для анонимных функций.

И анонимные, и стрелочные функции реализовали через класс Closure(PHP представляет анонимные функции как объекты с типом Closure. Класс содержит методы, которые разрешают управлять экземпляром анонимной функции).

Когда стрелочная функция работает с переменной, которую определили в родительской области, переменная неявно захватывается по значению. В следующем примере функции $fn1 и $fn2 ведут себя одинаково.

```php
<?php
$y = 1;
$fn1 = fn($x) => $x + $y;

// Эквивалентно получению переменной $y по значению:
$fn2 = function ($x) use ($y) {   return $x + $y;   };
var_export($fn1(3));
?>
```

1. *Синтаксис*:
   Стрелочная функция строится следующим образом:
   
```php
<?php
fn (параметры) => выражение;
?>
```
   
   Например:
   
```php
<?php
$sum = fn($a, $b) => $a + $b;
?>
```
   

2. *Замыкания*:
   Стрелочные функции автоматически используют переменные из внешней области видимости (замыкания). Это означает, что нет необходимости использовать ключевое слово use:
   
```php
<?php
$factor = 2;
$double = fn($n) => $n * $factor;
?>
```
   

3. *Однострочные функции*:
   Они ограничены одним выражением, что делает их очень удобными для простых операций. Результат этого выражения автоматически возвращается:
   
```php
<?php   $triple = fn($n) => $n * 3; // возвращает утроенное значение ?>
```
   

4. *Использование*:
   Стрелочные функции особенно полезны при работе с массивами и функциями высшего порядка, такими как array_map, array_filter и другими:
   
```php
<?php
$numbers = [1, 2, 3];
$squared = array_map(fn($n) => $n ** 2, $numbers);
?>
```
   

5. *Преимущества*:
   - Более читабельный и лаконичный код.
   - Упрощает работу с функциями, которые требуют анонимные функции.

В PHP существует несколько управляющих конструкций для включения файлов:

require – используется для обязательного подключения файла. Если файл не найден или произошла ошибка при его подключении, выполнение скрипта прерывается с фатальной ошибкой.

```php
require 'filename.php';
```
include – аналогичен require, но если файл не будет найден или произойдет ошибка при его подключении, то скрипт продолжит выполняться, выдав предупреждение (warning).

```php
include 'filename.php';
```
require_once – работает так же, как и require, за исключением того, что если файл уже был включен ранее, он не будет включен повторно.

```php
require_once 'filename.php';
```
include_once – аналогично include, но также предотвращает повторное включение одного и того же файла.

```php
include_once 'filename.php';
```
Основные различия между этими конструкциями:
require и include:
require останавливает выполнение программы при отсутствии файла, а include продолжает работу, выводя предупреждение.
once версии (require_once и include_once) предотвращают повторное подключение файла, что полезно для предотвращения дублирования кода и ошибок при многократном включении одних и тех же функций или классов.

### **Классы и объекты. Атрибуты классов. Модификатор доступа public.**

Каждое определение класса начинается **с ключевого слова class**, затем идёт **имя класса**, а потом **пара фигурных скобок**, **в которых** определяют **свойства и методы** класса.

Для класса разрешается выбирать произвольное название, при условии, что это **не зарезервированное слово**(break,die,switch(ключевые слова), directory (предопределенные классы), PHP_VERSION(предопределенные константы),int, true,enum,iterable) языка PHP.

Допустимое название класса начинается с буквы или подчеркивания, за которым идёт произвольное количество букв, цифр или символов подчеркивания.

Классы содержат константы, переменные, которые в классах называют свойствами, и функции, которые в классах называют методами.

```php
class SimpleClass
{
	// Объявление свойств
	public $var = 'значение по умолчанию';
	// Объявление метода
	public function displayVar(){
		echo $this->var;
	}
}
```

В PHP атрибуты классов позволяют добавлять метаданные к классам, методам, свойствам и другим элементам кода. Это позволяет помечать элементы программы для того, чтобы они могли быть обработаны различными инструментами, такими как фреймворки, библиотеки или компиляторы. Атрибуты — про добавление дополнительной информации и конфигурацию. Интерфейсы реализуют только классами, тогда как атрибуты разрешается нацеливать на методы, функции, параметры, свойства и константы классов. Поэтому атрибуты — существенно более гибкий механизм, чем интерфейсы.

**1. Извлечение атрибутов**

Для получения информации об атрибутах, примененных к классу, методу или свойству, используется механизм рефлексии. В PHP для этого существуют специальные классы, такие как ReflectionClass, ReflectionMethod, ReflectionProperty и другие.

**Пример: Получение атрибутов класса**

Предположим, у нас есть следующий класс с атрибутом:

```php
[MyAttribute]
class MyClass {
    // ...
}
```

Чтобы получить информацию об этом атрибуте, мы можем воспользоваться объектом ReflectionClass:

```php
$reflection = new ReflectionClass(MyClass::class);
$attributes = $reflection->getAttributes();
foreach ($attributes as $attribute){
	echo '__Класс_ _имеет_ _атрибут__: ', $attribute->getName(), "\n";
}
```

Метод getAttributes() возвращает массив объектов ReflectionAttribute, каждый из которых представляет собой экземпляр атрибута, применяемого к данному классу. Метод getName() возвращает имя атрибута.

**Пример: Получение атрибутов метода**

Если атрибут применяется к методу:

```php
class MyClass {
    [AnotherAttribute]
    public function myMethod() {
        //_ _Код_ _метода_
}
}
```

Тогда для получения информации об атрибутах метода можно использовать объект ReflectionMethod:

```php
$reflection = new ReflectionMethod(MyClass::class, 'myMethod');
$attributes = $reflection->getAttributes();
foreach ($attributes as $attribute) {
    echo '__Метод_ _имеет_ _атрибут__: ', $attribute->getName(), "\n";
}
```

**Модификатор доступа** **Public**

Доступ к общедоступным членам класса, — которые объявили с ключевым словом public, — разрешается из любой области видимости. Модификатор доступа public в PHP означает, **что свойство или метод класса доступны везде: внутри самого класса, в его дочерних классах и снаружи класса**. Это **самый открытый уровень видимости**, позволяющий любому коду обращаться к этим членам класса.

Пример использования:

```php
class Example {
    public $name;
    public function sayHello() {
        return "Привет!";
    }
}
// Доступ к публичному свойству и методу вне класса_
$example = new Example();
echo $example->name = "Мир"; // Можно изменить значение свойства
echo $example->sayHello(); // Вызвать публичный метод
```

### Конструкторы

*3. Конструкторы:*
- Конструктор — это специальный метод класса, который вызывается автоматически при создании объекта. Он используется для инициализации свойств объекта.
- Определяется с помощью метода __construct().

Пример:

```php
class Animal {
    public $name;
    public function __construct($name) {
        $this->name = $name;
    }
}
$cat = new Animal("Мурка");
echo $cat->name; // Вывод: Мурка
```
### Деструкторы

*4. Деструкторы:*
- Деструктор — это метод, который вызывается автоматически при уничтожении объекта. Он может использоваться для выполнения завершающих операций, таких как освобождение ресурсов.
- Определяется с помощью метода __destruct().

Пример:
```php
class Animal {
    public function __destruct() {
        echo "Объект уничтожен.";
    }
}
$dog = new Animal();
// Объект $dog будет уничтожен, когда выйдет из области видимости.
```

### Классы и объекты. Наследование классов

#### 1. Классы и объекты
- *Класс*: это шаблон или чертеж для создания объектов. Класс определяет свойства (переменные) и методы (функции), которые будут доступны объектам этого класса.
- *Объект*: это экземпляр класса, имеющий свои собственные данные. Объекты создаются с использованием ключевого слова new.

*Пример создания класса и объекта*:
```php
<?php
class Car {
	public $color;
    public $model;
    public function drive() {
        return "The car is driving.";
    }
}
$myCar = new Car();
$myCar->color = "Red";
$myCar->model = "Toyota";
echo $myCar->drive(); // Вывод: The car is driving.
?>
```
#### 2. Свойства и методы
- Свойства определяют состояние объекта (атрибуты).
- Методы описывают поведение объекта (функции).

#### 3. Модификаторы доступа
- public: доступен из любого места.
- protected: доступен только в классе и его наследниках.
- private: доступен только в самом классе.

#### 4. Наследование классов
- Наследование позволяет создавать новый класс на основе существующего, унаследуя его свойства и методы.
- Используется ключевое слово extends.

*Пример наследования*:
```php
<?php
class Vehicle {
	public function start() {
        return "Vehicle started.";
    }
}

class Car extends Vehicle {
    public function honk() {
        return "Beep!";
    }
}
$myCar = new Car();
echo $myCar->start(); // Вывод: Vehicle started.
echo $myCar->honk();  // Вывод: Beep!
?>
```
#### 5. Переопределение методов
- Методы родительского класса могут быть переопределены в дочернем классе.

*Пример переопределения*:

```php
<?php
class Vehicle {
    public function start() {
        return "Vehicle started.";
    }
}

class Car extends Vehicle {
    public function start() {
        return "Car started.";
    }
}
$myCar = new Car();
echo $myCar->start();
// Вывод: Car started.
?>
```

#### 6. Полиморфизм
- Полиморфизм позволяет использовать методы, которые определены в родительском классе, в дочерних классах.

#### 7. Абстрактные классы и интерфейсы
- *Абстрактный класс*: класс, который не может быть инстанцирован. Он может содержать как абстрактные, так и конкретные методы.
- *Интерфейс*: определяет набор методов, которые класс должен реализовать, но не содержит их реализации.

*Пример абстрактного класса*:

```php
<?php
abstract class Animal {
    abstract public function makeSound();
}
class Dog extends Animal {
    public function makeSound() {
        return "Bark!";
    }
}
$dog = new Dog();
echo $dog->makeSound();
// Вывод: Bark!
?>
```

### Классы и объекты. Автоматическая загрузка классов.

Функция spl_autoload_register() в PHP позволяет регистрировать одну или несколько функций автозагрузки классов. Это значит, что вместо явного включения файлов с классами через require или include, PHP автоматически загрузит нужный файл класса, когда он впервые будет использован в программе.

Как работает spl_autoload_register()

Когда вы используете новый класс в вашем коде, PHP сначала проверяет, загружен ли этот класс. Если нет, то вызывается зарегистрированная функция автозагрузчика, которая пытается найти и включить соответствующий файл с определением этого класса.

Предположим, все ваши классы находятся в папке classes/, и имена файлов совпадают с именами классов:

```php
function myAutoloader($className){
	$filePath = __DIR__ . '/classes/' . $className . ' .php';
	if(file_exist($filePath)){
		require_once $filePath;
	}
}
spl_autoload_register('myAutoload');
```

Теперь, когда вы создадите объект какого-либо класса, например, new MyClass();, PHP автоматически попытается загрузить файл /classes/MyClass.php.
