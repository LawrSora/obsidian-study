# Модель структурированного проектирования

**Описание**:  
Процесс разработки делится на последовательные этапы: анализ требований, проектирование, реализация, [[Тестирование ПО|тестирование]], внедрение. Акцент на формализацию требований и документирование.  
**Применение**:
Подходит для проектов с фиксированными требованиями (например, государственные системы).

> [!NOTE] **Методологии**:
> 
> - **[[Каскадная (waterfall) модель разработки ПО|Waterfall]] (водопадная модель)**: Жесткая последовательность этапов.
>     
> - **Spiral (спиральная модель)**: Итеративное развитие с оценкой рисков.
