> [!info] Определение
Тестирование [[Программное обеспечение(ПО)|ПО]] - это процесс проверки его на соответствие определённым требованиям и оценки качества. Оно проводится для обнаружения ошибок, дефектов и несоответствий в работе программы, а также для проверки соответствия программы требованиям заказчика.

### 1. По запуску кода на исполнение

| **Вид**                       | **Описание**                 | **Пример**                                     |
| ----------------------------- | ---------------------------- | ---------------------------------------------- |
| [[Статическое тестирование]]  | Проверка без выполнения кода | Анализ документации, проверка синтаксиса кода  |
| [[Динамическое тестирование]] | Проверка с запуском кода     | Функциональные тесты, нагрузочное тестирование |

---

### 2. По доступу к коду и архитектуре

| **Метод**                              | **Доступ к коду** | **Основа**                        | **Пример**                                      |
| -------------------------------------- | ----------------- | --------------------------------- | ----------------------------------------------- |
| [[Метод белого ящика\|Белого ящика]]   | Полный            | Знание реализации                 | Анализ кода поля ввода, проверка всех сценариев |
| [[Метод серого ящика\|Серого ящика]]   | Частичный         | Комбинация внешнего и внутреннего | Тестирование API через Postman                  |
| [[Метод чёрного ящика\|Черного ящика]] | Отсутствует       | Требования/спецификации           | Тестирование по макетам дизайна                 |

---

### 3. По степени автоматизации

| **Вид**                                                 | **Описание**                        | **Пример**                                     |
| ------------------------------------------------------- | ----------------------------------- | ---------------------------------------------- |
| [[Ручное тестирование\|Ручное]]                         | Выполняется тестировщиком вручную   | Проверка UI, исследовательское тестирование    |
| [[Автоматизированное тестирование\|Автоматизированное]] | Выполняется скриптами/инструментами | Написание тестов на JavaScript для авторизации |

---

### 4. По уровню детализации приложения

| **Вид**                                         | **Объект проверки**         | **Пример**                                            |
| ----------------------------------------------- | --------------------------- | ----------------------------------------------------- |
| [[Модульное тестирование\|Модульное]]           | Отдельный компонент/функция | Тест кнопки "Поделиться поездкой"                     |
| [[Интеграционное тестирование\|Интеграционное]] | Взаимодействие компонентов  | Проверка связи карты и поля адреса в такси-приложении |
| [[Системное тестирование\|Системное]]           | Приложение целиком          | Сквозной сценарий покупки товара                      |

---

### 5. По степени важности функций

| **Вид**                                                | **Цель**                                                   | **Пример**                                |
| ------------------------------------------------------ | ---------------------------------------------------------- | ----------------------------------------- |
| [[Дымовое тестирование\|Дымовое (Smoke)]]              | Проверить критически важный функционал                     | Доступность главной страницы, авторизация |
| [[Тестирование критического пути\|Критического пути]]  | Проверить основные сценарии пользователя                   | Оформление заказа в интернет-магазине     |
| [[Расширенное тестирование\|Расширенное тестирование]] | Проверить всю функциональность (включая низкоприоритетную) | Тестирование ссылок в подвале сайта       |

---

### 6. По принципам работы с приложением

| **Вид**                                 | **Подход**                             | **Пример**                           |
| --------------------------------------- | -------------------------------------- | ------------------------------------ |
| [[Позитивное тестирование\|Позитивное]] | Проверка корректных данных/сценариев   | Ввод валидного email при регистрации |
| [[Негативное тестирование\|Негативное]] | Проверка некорректных данных/сценариев | Ввод текста в поле "Номер телефона"  |

---

### 7. Связанные с изменениями в коде

| **Вид**                                          | **Когда применяется**       | **Пример**                                               |
| ------------------------------------------------ | --------------------------- | -------------------------------------------------------- |
| [[Тестирование сборки]]                          | Перед релизом версии        | Проверка стабильности после сборки                       |
| [[Санитарное тестирование\|Санитарное (Sanity)]] | После изменений в коде      | Быстрая проверка ключевых функций модуля                 |
| [[Регрессионное тестирование\|Регрессионное]]    | После доработок/исправлений | Проверка, что исправление бага не сломало другие функции |
